## Locked algo (per the brainstorm, complete)

Don't relitigate any of these. They're decided.

- **R1 / R1a**: per-set base XP = `effective_load × repMult(reps) × reps`. effective_load uses **IPF GL Points normalization** (replaced linear scaling on 2026-05-06). For BW exercises, includes `REFERENCE_BW × bw_coefficient + entered_weight × norm_factor`.
- **R2**: total XP per set = additive multiplier stack: `base × (1 + consistency + class + prestige + holiday)`. Class is one of compound 1.5 / king 1.75 / iso 1.2.
- **R5–R5c**: 21-tier teeth-grip ladder (RELAXED → GRITTED). Counter = lifetime cumulative count of 100%-completion sessions (no decay). Pacing curve front-loaded, 100 sessions to peak. Geometric multiplier curve ×1.00 → ×3.00.
- **R7**: tier-up flourish (kanji, color, animation) on tier crossings. Specifics not designed yet.
- **R8 / R8a**: completion-threshold partial credit. <50% sets done = combo fizzles (consistency mult 0×); 50–99% scales linearly; 100% full credit + tier counter ticks. End-of-day reckoning model — consistency contribution computed at session close.
- **R9**: prestige loop. GRITTED + 20 sessions → Ascend option (optional, can hold). Ascend resets counter to RELAXED + grants Galaxy-Spiral ribbon. Each ribbon = +0.10× prestige_mult permanently. 120 sessions per ribbon cycle.
- **R10 / R10a**: region weight model (60% primary / 40% secondary, dual-semantics region map: chest+quads → FRONT, back+hams → BACK, shoulders+biceps+triceps+forearms → ARMS, abs → CORE, glutes+calves → LEGS).
- **R11**: legacy band thresholds — retired. Don't gate stars on raw region share.
- **R12**: compound = top 2 regions × 1★ each.
- **R12b**: King compound = top 3 regions × 1★ each.
- **R12c**: star-earning floor — set must hit `relative_load >= 0.75 × heavy_lift_threshold` to earn region stars. XP path is unaffected.
- **R13**: isolation = single top region × 2★.
- **R14**: auto-classify rule + `ISOLATION_OVERRIDE` set for the 13 hand-flagged isolation exceptions.
- **R16**: holiday multiplier — 12 days/yr per user, three tiers (×1.5 birthday/Christmas/NYD; ×1.0 Jul 4 / Thanksgiving; ×0.5 other 7 federal).
- **R17**: active-page nav stays clean — no constant multiplier display.
- **R18 / R18a**: per-set cinematic reveal (~1.2-1.5s). HEAVY LIFT bonus line shown when `combined_factor > 1` (norm_factor × (1 + heavy_lift_x)). Asymmetric UI by design — light users feel rewarded; heavy users see normal cinematic.
- **R19**: region growth surfaces visually via existing transmutation-circle on stats page.
- **R20 / R20a**: profile page = identity tag with current tier name + ribbon row. Stats page = progress bar + cumulative count + ribbon history.

## What still needs implementation (i.e., what `/ce:plan` should plan)

The algo is fully specified but not coded. A worker (or set of workers) will need to:

1. **Profile schema** — add `user_bodyweight` (lb) and `user_sex` (male/female default male) fields. Onboarding flow captures `user_bodyweight` first time. Settings page allows editing.
2. **BW unset gate** — modal forces user to enter bodyweight before saving the first BW-coefficient set.
3. **Algo runtime** — port the locked spec to JS:
   - `ipfGL(bw_kg, params)` function
   - `bodyweightNormFactor(user_BW_lb, sex)` function
   - `effectiveLoad(set, exercise, user)` function
   - `calculateSetXP(set, exercise, user, runtime_state)` returning the cinematic stack
4. **Tier counter persistence** — store cumulative 100%-session count + active tier per profile.
5. **End-of-day reckoning** — when the last planned set of a session is logged or the day rolls over, compute completion% and apply consistency contribution + tier-counter tick.
6. **Prestige unlock + ribbon storage** — track when GRITTED + 20 hit; expose Ascend UI; persist ribbon count.
7. **Cinematic UI** — ~1.2-1.5s sequential reveal on per-set log animation. R18a HEAVY LIFT line conditionally visible. Inactive multipliers skip.
8. **Star track** — region-star awards per set, gated by R12c floor. Star animations on the existing transmutation-circle on stats page.
9. **Profile + Stats UI** — identity tag (R20), progress bar / cumulative count / ribbon row (R20a).
10. **Holiday detection** — date-based check at workout time.
